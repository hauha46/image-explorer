# ==========================================================================
# image-explorer backend requirements (HARDENED PIN SET)
# ==========================================================================
# The pins below are NOT suggestions; they are the versions that make every
# synthesizer (SEVA, ViewCrafter, DUSt3R, DepthPro) work together on a
# torch-2.4.1 + CUDA-12.1 pod.  Each strict pin is here because loosening
# it caused an A100 sweep to fail mid-run.  See scripts/setup_runpod.sh
# comments for the exact bug each one fixes.
#
# Install order (handled by setup_runpod.sh):
#   1. torch / torchvision from the PyTorch cu121 index
#   2. this requirements.txt
#   3. pytorch3d from source (needs torch already present)
#   4. flash-attn (optional)
#
# If you bump any pin below, run `bash scripts/run_all.sh` end-to-end on a
# fresh pod before merging -- the failure modes do not surface until deep
# inside ViewCrafter's diffusion loop.
# ==========================================================================

# --- PyTorch (installed separately from cu121 index; see setup_runpod.sh) -
# Pinning is handled out-of-band because the CUDA wheel URL is not
# representable in requirements.txt.  If you run `pip install -r` directly,
# you will get CPU wheels.
#
# WARNING: DO NOT add `torch` / `torchvision` here, even with a version
# pin.  If you do, `uv pip install --force-reinstall -r requirements.txt`
# will either (a) fetch the CPU-only PyPI wheels, or (b) if any downstream
# pin is loosened, re-resolve torch to the latest CUDA 13 build, which
# fails to load on RunPod A100 pods whose driver is CUDA 12.x.  This has
# already bitten us once.
#
# Instead, torch/torchvision are installed in setup_runpod.sh §6 via:
#   uv pip install torch==2.4.1 torchvision==0.19.1 \
#       --index-url https://download.pytorch.org/whl/cu121

# --- ViewCrafter / DynamiCrafter compatibility pin set -------------------
# Each of these was pinned after a specific crash during an A100 sweep.
#   open_clip_torch 2.24.x  -> ViewCrafter's lvdm/condition.py checks
#       `self.model.visual.input_patchnorm`, an attribute that was renamed
#       in 2.24+.  2.20.0 is the last version with the old name.
#   pytorch_lightning 2.x   -> ViewCrafter inherits from LightningModule;
#       the 1.x->2.x training-API rewrite breaks VC's checkpoint loading.
#   av 12+                  -> torchvision.io.write_video assigns
#       `frame.pict_type = "NONE"` (string); av >=12 made pict_type strict
#       int/enum-only.  av 10 won't compile against Cython 3; av 11 does.
#   moviepy 2.x             -> 2.0 was a full API rewrite; VC uses 1.x APIs.
#   pillow 10+              -> removed Image.ANTIALIAS, which DUSt3R's
#       `center_crop_pil_image` still references.
#   numpy 2.x               -> removed np.bool / np.int / np.float aliases;
#       a lot of 2023-era CV code in the vendor trees still uses them.
open-clip-torch==2.20.0
pytorch-lightning>=1.9.0,<2.0
av==11.0.0
moviepy>=1.0,<2.0
